### **Определения**

| Термин | Определение | Формат | Правило верификации | Синоним |
| :--- | :--- | :--- | :--- | :--- |
| **Пользователь** | Зарегистрированное лицо, обладающее учётной записью и правами на отправку/получение сообщений. | Строка (имя, логин), UUID для ID | Уникальный идентификатор должен существовать в базе. | Абонент |
| **Идентификатор пользователя (UserID)** | Уникальная сущность, однозначно определяющая пользователя в системе. | UUID или числовой ID | Не должен повторяться. | UID |
| **Чат** | Логически оформленное пространство для переписки нескольких или одного пользователя. | Строка (название), UUID | ID чата должен быть уникальным. | Диалог, Комната |
| **Личный чат** | Чат между двумя пользователями. | UUID чата, два UID | Оба участника должны существовать. | Диалог |
| **Групповой чат** | Чат, содержащий двух и более участников. | UUID чата, список UID | Минимум два участника. | Группа |
| **Сообщение** | Единица текстовой информации, отправляемая пользователем. | Текст, метка времени, ID автора | Поле автора не пустое, допустимый размер. | Msg |
| **Временная метка (Timestamp)** | Время отправки сообщения. | ISO-8601 | Должно быть корректным временем. | Time |
| **Вложения (Attachment)** | Файл, передаваемый в составе сообщения. | Имя файла, размер, MIME-тип | Размер не превышает системных лимитов. | File, Attachment |
| **Статус доставки** | Текущее состояние сообщения: отправлено, доставлено, прочитано. | Enum | Должно принадлежать допустимым статусам. | Delivery State |
| **Контакт** | Элемент списка друзей/собеседников пользователя. | UID контакта | Контакт должен существовать в системе. | Friend |
| **Устройство** | Телефон, ПК или планшет, через который пользователь входит в систему. | Строка (тип), UUID | Должно быть связано с корректным UserID. | Client Device |
| **Сессия** | Активное подключение пользователя к системе. | токен, время жизни | Токен должен быть действительным. | Session |
| **Авторизация** | Проверка личности пользователя системой. | Логин, пароль/код | Учетная запись должна состоять в системе. | Login |
| **Токен авторизации** | Уникальный ключ, подтверждающий авторизованное состояние. | JWT/строка | Не должен быть просрочен. | Auth Token |
| **Сервер сообщений** | Центральный компонент, принимающий, обрабатывающий и пересылающий сообщения. | – | Должен быть доступен и интегрирован с БД. | Message Server |
| **Уведомление** | Сигнал о новом событии (сообщении, приглашении и т.п.). | Push/всплывающее сообщение | Должно быть связано с корректным событием. | Notification |
| **Модератор** | Пользователь с расширенными правами управления чатом. | UID + набор разрешений | Должен быть участником группы. | Admin (частично) |
| **Приглашение** | Специальное сообщение о вступлении в чат. | Структура данных (от кого, кому, куда) | Должно содержать валидный ChatID. | Invite |
| **Черный список** | Список пользователей, которым запрещено взаимодействовать с данным пользователем. | Массив UID | UID должен существовать. | Blocklist |

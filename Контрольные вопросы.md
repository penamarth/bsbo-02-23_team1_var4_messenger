## **Контрольные вопросы**

### 1. Что такое UML? Основные диаграммы

**UML** (Unified Modeling Language) — язык для рисования схем программ. Помогает понять систему до её написания.

**Основные диаграммы:**

**Диаграммы структуры:**
- **Диаграмма классов** — какие классы есть в программе и как они связаны
- **Диаграмма компонентов** — из каких частей состоит программа
- **Диаграмма развертывания** — на каких серверах что работает

**Диаграммы поведения:**
- **Диаграмма прецедентов** — кто и что может делать в системе
- **Диаграмма последовательностей** — кто кому и какие сообщения отправляет
- **Диаграмма состояний** — как меняется состояние объекта
- **Диаграмма деятельности** — блок-схема процесса

### 2. Элементы диаграммы классов

**1. Класс** — рисуется прямоугольником с тремя отделами:
```

│   User          |  Название класса

│ - id: String    │  Поля (атрибуты)
│ + name: String  │

│ + login()       │  Методы
│ - validate()    │

```

**2. Атрибуты** — переменные класса. Перед ними ставят:
- `+` — публичный (видно всем)
- `-` — приватный (видно только классу)
- `#` — защищенный (видно классу и наследникам)

**3. Методы** — функции класса. Формат: `имя(параметры): тип_возврата`

**4. Связи между классами:**
- **Ассоциация** — обычная связь (сплошная линия)
- **Наследование** — стрелка с пустым треугольником 
- **Реализация интерфейса** — пунктир со стрелкой 
- **Агрегация** — ромб не закрашен — "часть может жить без целого"
- **Композиция** — ромб закрашен — "часть умирает с целым"
- **Зависимость** — пунктир со стрелкой 

**5. Кратность** — цифры на концах связей:
- `1` — ровно один
- `0..1` — ноль или один
- `*` — много (0 или больше)
- `1..*` — один или больше
